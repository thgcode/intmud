Variáveis dinâmicas de personagens, salas e itens:

Todos os personagens e salas têm uma lista de variáveis dinâmicas
(podem ser criadas e apagadas) contidas na seguinte variável:
textovar var

Os dois primeiros caracteres das variáveis de var indicam o tipo:
a_ = agenda (1=viu, 2=pegou)
d_ = variável que é apagada quando o objeto muda de dono
e_ = nome da classe que contém o efeito
     tem efeito somente quando o item ou personagem é criado
f_ = nome de um modificador
j_ = variável que é apagada quando é carregado um jogo salvo
h_ = habilidade
i_ = idioma
m_ = magia
o_ = item classificado como "outros"
r_ = rastro do jogador; exemplo:
     var.r_João@=15 na sala é o rastro de João; desaparece em 1.5 segundo
     var.r_João@=15 em PNJ, vai atacar se encontrar João em até 1.5 segundo
t_ = atalho (vide comando atalhos)
u_ = usuário ignorado
     em salas (casas) = jogador que tem permissão para pegar e soltar itens
z_ = variável não visível ao jogador


Variáveis usadas no MUD:
z_aulas = pontos de aula (quantas aulas ainda pode fazer)
z_captura = nome do item que o jogador escolheu para capturar animais
z_desc = descrição criada por um jogador; vide comandos pdesc e sdesc
z_dono = nome do dono original (quem capturou o animal)
z_email = e-mail do jogador (somente nos personagens de jogadores)
z_liquido = líquido que tem dentro (quantidade e nome do líquido)
    Exemplo, "10 água" significa 10ml de $liq_água
    Após mudar a quantidade de líquido, chamar a função ajustapeso do item
z_mdesc = descrição criada com o comando mdesc
z_moedas = quantas moedas o personagem, item ou sala possui
z_pnivel = nível do animal quando foi capturado
z_salafim = sala para onde vai quando retorna da morte
z_teleentra = mensagem definida com o comando teleentra
z_telesai = mensagem definida com o comando telesai
z_tipo1 = raças que o jogador já completou (uma linha por raça)
z_tipo2 = classes que o jogador já completou (uma linha por classe)
z_tipo3 = subclasses que o jogador já completou (uma linha por subclasse)
z_titulo = título criado por um jogador; vide comandos ptitulo e rtitulo


Variáveis usadas em jogsock:
z_responder = para quem está respondendo
z_telepatia = quem enviou a última mensagem telepática


Variáveis usadas em indiceobj e indiceitem:

São usados os seguintes textos em indiceobj/indiceitem:
"ip " + endereço IP
    Contém a lista de usuários com esse endereço IP
    Variável jogoservip:endereco, arquivo misc/jogoserv.int
"bs " + endereço do chat + " " + nome da sala no chat
    Uma sala, quando o MUD é ativado como bot em um chat
    Variável jogobot_sala:nome, arquivo misc/jogobot.int
"bu " + endereço do chat + " " + nome do usuário no chat
    Um usuário, quando o MUD é ativado como bot em um chat
    Variável jogobot_usr:j_nome, arquivo misc/jogobot.int
"un " + nome codificado (nome do arquivo SAV sem o .SAV)
    O objeto responsável pela comunicação com o jogador
    Variável jogsock:cnome, arquivo obj/jogsock.int
"pn " + nome codificado (nome do arquivo SAV sem o .SAV)
    Personagem do jogador
    Variável jogador:cnome, arquivo obj/jogador.int
"cj " + nome codificado + " " + prefixo da área
    Casa do jogador (nome codificado corresponde ao arquivo SAV do jogador)
    Variável casa:nomejog, arquivo obj/casa.int
